# Task Detail 页面重设计

## 目标
将 Task Detail 页面从当前的平铺卡片布局改造为 Dashboard 仪表盘风格，提升信息密度和可读性。

## 当前痛点
1. 头部信息单薄，缺少时间线和上下文
2. 统计卡片缺乏视觉层次，颜色单一
3. 数据准备和分析步骤割裂，看不到完整 pipeline
4. 结果展示用 Table + JSON stringify，可读性极差
5. 整体布局平铺，缺乏信息层级

## 设计方案

### 1. 任务概览头部
- 左侧：任务名称（大标题）、描述（如有）、创建时间 / 更新时间
- 右侧：状态 Badge（加大）+ 总进度百分比 + 操作按钮

### 2. 关键指标行（KPI Cards）
4 个紧凑指标卡片，带状态色和图标：
- 总任务（默认色）
- 已完成（绿色 + 对勾图标）
- 失败（红色 + 警告图标）
- 进行中（蓝色 + 加载图标）

### 3. Pipeline 时间线（纵向）
将数据准备和分析步骤融合为一条垂直时间线：

```
● 数据准备          [========>] 100%
│
● 步骤1: 情感分析    [========>] 100%
│
○ 步骤2: 风格分析    [==>      ] 23%  ▶ 进行中
│
○ 步骤3: 标签提取    [         ] 0%    ○ 待开始
```

- 节点圆点：灰(待开始)、蓝脉冲(进行中)、绿(完成)、红(失败)
- 右侧迷你进度条显示完成百分比
- 点击节点展开该阶段的汇总统计

### 4. Pipeline 矩阵表（核心）
以**帖子为行，阶段为列**的矩阵视图：

| 帖子 | 数据准备 | 步骤1:情感分析 | 步骤2:风格分析 |
|------|----------|----------------|----------------|
| post_abc | ✅ | ✅ | 🔄 |
| post_def | ✅ | ⚠️ | ⏳ |
| post_ghi | 🔄 | ⏳ | ⏳ |

- 单元格状态：✅ 完成 / 🔄 进行中 / ⚠️ 失败 / ⏳ 待开始
- 点击单元格展开该帖子在该阶段的详细结果（Modal）
- 表头固定，步骤多的时候支持横向滚动
- 行可点击跳转到帖子详情

## 技术实现要点
- 复用现有 shadcn/ui 组件：Card、Badge、Progress、Skeleton
- 新增：Timeline 组件（纵向）、PipelineMatrix 组件
- 结果 Modal 复用 PostLibrary 的详情展示逻辑
- 状态颜色统一使用已有的 statusVariantMap 映射
